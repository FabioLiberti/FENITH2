# Security Policy

## Scope

FENITH is an academic meta-repository aggregating Federated Learning research for European Digital Healthcare. Several submodules deal with privacy-preserving techniques, differential privacy, and secure aggregation applied to sensitive healthcare data.

## Reporting a Vulnerability

If you discover a security issue in any FENITH component, please report it responsibly:

1. **Do not** open a public GitHub issue.
2. Contact the author directly via [www.fenith.org](https://www.fenith.org/) or through the university email listed on [ORCID](https://orcid.org/0000-0003-3019-5411).
3. Include a description of the vulnerability, steps to reproduce, and the affected repository/submodule.

You will receive an acknowledgement within 7 working days.

## Supported Versions

| Version | Supported |
|---|---|
| Latest release | Yes |
| Older releases | Best effort |

## Responsible Disclosure

We follow a coordinated disclosure process. Please allow reasonable time for a fix before any public disclosure.
